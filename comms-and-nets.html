Communication Methods

Serial vs Parallel Transmission

Serial transmission sends bits sequentially over a single channel. Parallel transmission sends multiple bits simultaneously across multiple channels.

  • Advantages of serial over parallel:
    • Lower cost and fewer wires.
    • Reduced crosstalk and skew problems at high speeds.
    • Better for long-distance links (less timing/alignment issues).

Synchronous vs Asynchronous Transmission

Synchronous
Data is sent in a continuous stream with sender and receiver sharing a clock or using embedded clocking. Efficient for high-speed bulk transfers (e.g., network links).
Asynchronous
Data is sent in discrete units (usually bytes) with no shared clock; timing is recovered from start/stop bits. Simpler for intermittent or low-latency transfers (e.g., serial console).

Start and Stop Bits (Asynchronous)

Start and stop bits frame each data byte so the receiver can detect the beginning and end of transmission when no shared clock exists:

  • Start bit: indicates the start of a data unit (usually a low level).
  • Stop bit(s): indicate the end of the byte (one or two high-level bits); allow the receiver to re-synchronize between bytes.

Communication Basics

Baud rate
The number of signal changes (symbols) transmitted per second, measured in bauds.
Bit rate
The number of bits transmitted per second (bps).
Bandwidth
The range of frequencies available for signal transmission, often expressed in Hz; in networking it can mean capacity (bps).
Latency
The time delay between sending and receiving data (often measured in ms).
Protocol
A set of rules that define how data is formatted, transmitted, received, and interpreted between devices.

Baud rate vs Bit rate

Baud rate counts symbols per second. If each symbol encodes more than one bit (using modulation or multi-level signaling), bit rate can exceed baud rate:

  • bit rate = baud rate × bits per symbol

Bit rate and Bandwidth

Higher channel bandwidth (frequency range) typically allows higher bit rates, but efficiency depends on modulation, noise, and channel quality.

Networking — Topology

Physical Star Topology

Devices connect to a central hub or switch using individual links. Physical layout resembles a star.

  • Failure of a single peripheral doesn't break the whole network.
  • Central device is a single point of failure.

Logical Bus Topology

Devices share a single broadcast medium logically: messages sent on the bus are visible to all nodes.

  • Often implemented over physical star (e.g., Ethernet switched networks operate logically as a shared medium when using hubs or with broadcast domains).
  • Collision handling (CSMA/CD historically) and addressing are used to manage access.

Difference and Operation

Physical topology describes cabling and device placement; logical topology describes how data flows. Example: a switched Ethernet is physically star but can behave like a logical bus for broadcasts.

Networking Between Hosts

Peer-to-Peer Networking

Hosts have equal roles and can share resources directly. Useful for small workgroups or ad-hoc file sharing.

Client-Server Networking

Clients request services from centralized servers. Suitable for scalable services, centralized management, and security (e.g., web, email, authentication).

WiFi (Wireless Networking)

Purpose

WiFi provides wireless LAN connectivity to enable network access without wired connections — mobility and easy deployment are primary benefits.

Required Components

  • Wireless Access Point (AP) or router
  • Client devices with wireless adapters
  • Backhaul (wired network or internet link)
  • Optional: controller, switches, and authentication servers

Security

  • WPA2/WPA3 for encryption and authentication
  • Use strong passphrases, enterprise authentication (802.1X) for larger deployments
  • Network segmentation and guest networks for isolation

CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance)

Stations listen before transmitting. If the medium is idle they transmit; if busy they wait a random backoff. Collisions are avoided (because wireless stations may not hear each other).

With RTS/CTS: a station sends RTS (Request to Send), AP or receiver replies CTS (Clear to Send). This reserves the medium and reduces hidden-node collisions.

Service Set Identifier (SSID)

The SSID is the network name broadcast by an AP. It's used by clients to discover and select which wireless network to join.
